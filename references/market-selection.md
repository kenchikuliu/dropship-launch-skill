# Market Selection

Use this file when the user has not committed to a launch country yet.

## Default Ranking Logic

Pick the first market based on friction, not just market size.

Key factors:

- language and copy burden
- payment setup
- shipping speed expectations
- tax and import friction
- product-market fit for the category

## Default Market Guidance

### United Kingdom

Best first default when:

- the user wants an English-speaking market
- products are light and simple
- the user is running direct-response tests without local stock

Why:

- simpler language setup
- easier first-wave product testing for office and home organization
- strong fit for straightforward Shopify + Meta tests

### United States

Best when:

- the user has local inventory, US 3PL access, or can handle import friction
- the product has higher margin
- the user wants the biggest upside and can support faster delivery expectations

Why:

- largest scale potential
- stronger fit for higher-margin products
- not the cleanest first test when using pure direct shipping

### Brazil

Best when:

- the user can localize in Portuguese
- the product category suits stronger local adaptation
- the user is ready for higher localization and payment complexity

Why:

- attractive market, but not the lowest-friction first launch

## Recommendation Pattern

If the user has:

- no local stock and wants the cleanest first test: recommend `United Kingdom`
- local stock or US 3PL: recommend `United States`
- strong localization ability and a reason to target Brazil: recommend `Brazil`

Do not present any market as universally best.
